# Story 2.4: PWA Shell — Manifest, Service Worker and Install Prompt

Status: done

## Story

As a user,
I want ONYX to be installable as a PWA with offline support,
So that I can use it like a native app and access it without internet.

## Acceptance Criteria

### AC-1: PWA Manifest Configuration

- Given the PWA manifest is configured
- When I inspect `public/manifest.json`
- Then it includes: name ("ONYX"), short_name ("ONYX"), icons (192px and 512px), theme_color (#000000), display ("standalone"), start_url ("/")

- Given the manifest is loaded
- When a browser visits the app
- Then the manifest link is present in the `<head>` (already set via layout.tsx metadata: `manifest: "/manifest.json"`)
- And the manifest is served with `Content-Type: application/manifest+json`

### AC-2: Service Worker Registration

- Given the app loads in a browser
- When I inspect the network tab or console
- Then a Service Worker is registered on first load (via `@serwist/next`)
- And the Service Worker runs as expected (no registration errors)

### AC-3: Custom Install Prompt (beforeinstallprompt)

- Given I am using a supported browser (Chrome, Edge, Samsung Internet)
- When the `beforeinstallprompt` event fires
- Then a custom install prompt is presented on the dashboard (dismissable, max 2 appearances)
- And the prompt shows an "Install ONYX" button with the app icon

- Given I dismiss the install prompt
- When I refresh and return later
- Then the prompt does not appear more than 2 times total

- Given I have already installed the app
- When subsequent page loads occur
- Then no install prompt is shown (not applicable)

### AC-4: Standalone Mode

- Given I install the app
- When I open it from the home screen
- Then it opens in standalone mode with no browser chrome visible (`display: "standalone"`)
- And the theme_color (#000000) is used for the splash screen and status bar

### AC-5: Offline Shell Caching

- Given the Service Worker is active and configured with a `networkFirst` or `cacheFirst` strategy for static assets
- When I go offline and reload the app
- Then the app shell (layout, navigation, cached content) is displayed from the cache
- And no browser offline error page is shown
- And the Service Worker precaches static assets (JS, CSS, fonts, icons) on install

### AC-6: Build Integrity

- Given the PWA configuration is complete
- When I run `npm run build`
- Then the build succeeds with no errors
- And the Service Worker is generated by `@serwist/next` during build

### @serwist/next v9.x Configuration

```typescript
// next.config.ts — withSerwist wrapper pattern
import withSerwist from "@serwist/next";
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  turbopack: {
    root: __dirname,
  },
};

export default withSerwist({
  swSrc: "src/app/sw.ts",
  swDest: "public/sw.js",
})(nextConfig);
```

```typescript
// src/app/sw.ts — Service Worker entry
import { defaultCache } from "@serwist/next/worker";
import { Serwist, type PrecacheEntry } from "serwist";

// eslint-disable-next-line @typescript-eslint/no-explicit-any
declare const self: any;

const serwist = new Serwist({
  precacheEntries: self.__SW_MANIFEST satisfies (PrecacheEntry | string)[],
  skipWaiting: true,
  clientsClaim: true,
  navigationPreload: true,
  runtimeCaching: defaultCache,
});

serwist.addEventListeners();
```

**Cache strategies used by defaultCache:**
- Navigation requests: Network First (fallback to cache)
- Static assets (JS/CSS): Stale-While-Revalidate
- Fonts: Cache First
- Images: Cache First (with expiration)
- API requests: Network Only (API should not be cached by SW — handled by TanStack Query)
